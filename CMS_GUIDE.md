# Decap CMS 사용 가이드

네이버 블로그처럼 웹에서 바로 포스트를 작성할 수 있습니다!

## 접속 방법

1. 블로그가 배포된 후 다음 URL로 접속:
   ```
   https://lebronisbest.github.io/admin
   ```

2. **GitHub로 로그인** 버튼 클릭

3. GitHub 계정으로 인증

## 포스트 작성 방법

### 1. 새 포스트 만들기

1. 좌측 사이드바에서 **"포스트"** 클릭
2. 우측 상단 **"New 포스트"** 버튼 클릭
3. 다음 항목 입력:
   - **제목**: 포스트 제목 (필수)
   - **부제목**: 부가 설명 (선택)
   - **날짜**: 발행 날짜 (자동 입력됨)
   - **작성자**: 작성자 이름 (기본: lebronisbest)
   - **태그**: 쉼표로 구분하여 입력 (예: 개발, Python, 데이터)
   - **썸네일**: 대표 이미지 업로드 (선택)
   - **내용**: 마크다운으로 작성

### 2. 마크다운 에디터 사용

**Rich Text** 모드와 **Markdown** 모드를 전환할 수 있습니다:

- **Rich Text**: 워드처럼 직관적으로 작성
- **Markdown**: 마크다운 문법으로 직접 작성

### 3. 이미지 추가

두 가지 방법이 있습니다:

**방법 1: 드래그 앤 드롭**
- 에디터에 이미지를 드래그해서 놓기

**방법 2: 이미지 버튼 클릭**
- 툴바의 이미지 아이콘 클릭
- 파일 선택 또는 드래그 앤 드롭
- 이미지가 자동으로 `/assets/img`에 업로드됨

### 4. 저장 및 발행

1. 우측 상단 **"Save"** 버튼 클릭
2. **"Publish now"** 클릭하면 즉시 발행
3. 몇 분 후 블로그에 자동으로 반영됨 (GitHub Pages 빌드 시간)

## 기존 포스트 수정

1. 좌측 사이드바 **"포스트"** 클릭
2. 수정하려는 포스트 클릭
3. 내용 수정
4. **"Save"** → **"Publish now"** 클릭

## 페이지 수정

1. 좌측 사이드바 **"페이지"** 클릭
2. **"소개 페이지"** 또는 **"홈페이지"** 선택
3. 내용 수정
4. **"Save"** → **"Publish now"** 클릭

## 초안 기능 (선택사항)

초안 기능을 사용하려면 `admin/config.yml`에서 다음과 같이 수정:

```yaml
publish_mode: editorial_workflow
```

이렇게 하면:
- **Draft**: 초안
- **In Review**: 검토 중
- **Ready**: 발행 준비 완료

단계별로 관리할 수 있습니다.

## 마크다운 문법 가이드

### 제목
```markdown
# 제목 1
## 제목 2
### 제목 3
```

### 강조
```markdown
**굵게**
*기울임*
~~취소선~~
```

### 링크
```markdown
[링크 텍스트](https://example.com)
```

### 이미지
```markdown
![이미지 설명](/assets/img/image.jpg)
```

### 리스트
```markdown
- 항목 1
- 항목 2
  - 하위 항목

1. 번호 항목 1
2. 번호 항목 2
```

### 코드
````markdown
인라인 코드: `code`

코드 블록:
```python
def hello():
    print("Hello, World!")
```
````

### 인용구
```markdown
> 인용구 내용
```

## 문제 해결

### 로그인이 안 돼요
- GitHub 계정으로 로그인했는지 확인
- 저장소(lebronisbest/lebronisbest.github.io)에 대한 권한이 있는지 확인
- 브라우저 쿠키/캐시 삭제 후 재시도

### 이미지 업로드가 안 돼요
- 파일 크기 확인 (5MB 이하 권장)
- 파일 형식 확인 (jpg, png, gif, webp 지원)
- 인터넷 연결 확인

### 저장 후 블로그에 반영이 안 돼요
- GitHub Pages 빌드 시간이 필요합니다 (보통 1-5분)
- GitHub 저장소의 Actions 탭에서 빌드 상태 확인
- 브라우저 캐시 삭제 후 새로고침

### 삭제한 이미지가 그대로 있어요
- 이미지는 `/assets/img` 폴더에 저장됨
- 사용하지 않는 이미지는 GitHub 저장소에서 수동으로 삭제 필요

## 팁

1. **자동 저장 없음**: 작성 중간중간 저장하세요
2. **프리뷰 활용**: 우측 프리뷰로 결과를 확인하세요
3. **이미지 이름**: 한글보다는 영문으로 저장하는 것이 좋습니다
4. **정기적 백업**: 중요한 글은 별도로 백업하세요

## 로컬 테스트 (선택사항)

로컬에서 CMS를 테스트하려면:

1. `admin/config.yml`에 다음 추가:
   ```yaml
   local_backend: true
   ```

2. 터미널에서 실행:
   ```bash
   npx decap-server
   ```

3. 브라우저에서 `http://localhost:8080/admin` 접속
